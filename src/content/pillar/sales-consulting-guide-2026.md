## Hook (เปิดด้วย problem ไม่ใช่ context)

เจ้าของ SME ที่จ่ายค่าที่ปรึกษาการขายไป ฿100,000+ แล้วได้สไลด์กลับมา 80 หน้า ไม่ใช่เคสรายแรก

## 1. ที่ปรึกษาการขาย ≠ Sales Trainer ≠ Coach

| ประเภท | ส่งมอบอะไร | เหมาะกับใคร |
|---|---|---|
| **Sales Trainer** | ความรู้ + แรงบันดาลใจ 1 วัน | ทีมเก่งอยู่แล้ว ต้อง refresh skill |
| **Sales Coach** | คำแนะนำ 1-on-1 ระยะยาว | ผู้บริหารที่ต้องการ accountability |
| **Sales Consultant** | ระบบ + framework ใช้งานได้ | SME ที่ทีมยังไม่ run เอง |

ปันคือประเภทที่ 3 — บทความนี้จะลึกในการเลือก consultant ไม่ใช่ trainer

## 2. 5 เกณฑ์ตัดสินใจ (เรียงตามความสำคัญ)

### เกณฑ์ที่ 1: เขาส่งระบบ หรือส่งสไลด์
**เช็คยังไง:** ขอตัวอย่าง deliverable ของ engagement ก่อนหน้า — ถ้าตัวอย่างเป็น PowerPoint อย่างเดียว = trainer ไม่ใช่ consultant

[expand: ตัวอย่างเปรียบเทียบจริง — slide vs spreadsheet template + Apps Script + workflow diagram]

### เกณฑ์ที่ 2: เขาทำ industry คุณมาก่อนไหม
**เช็คยังไง:** ถามชื่อ client ใน industry เดียวกัน ขนาดธุรกิจใกล้เคียง — ถ้าตอบไม่ได้ = generic consultant อาจไม่เข้าใจ context

[expand: ทำไม industry context สำคัญ + ยกตัวอย่าง pricing structure ใน B2B วัสดุก่อสร้าง vs B2B IT services]

### เกณฑ์ที่ 3: เขาทำเองหรือ outsource
**เช็คยังไง:** ถาม "ใครจะเป็นคนทำ workshop / build system จริงๆ" — ถ้าเป็นทีมงาน junior = คุณกำลังจ่าย senior ราคา junior ส่ง

[expand: หลักการ pricing + risk transfer]

### เกณฑ์ที่ 4: เขามี skin in the game ไหม
**เช็คยังไง:** เขาเสนอ payment structure แบบไหน — fixed + 100% upfront, หรือ retainer ไม่มี deliverable ชัด, หรือ % of revenue

[expand: pros + cons แต่ละ structure + ปันเลือกแบบไหนและทำไม]

### เกณฑ์ที่ 5: เขามี handover plan ไหม
**เช็คยังไง:** ถาม "หลังจบ engagement ทีมเรา maintain ระบบเองได้ไหม" — ถ้าตอบ "ต้องจ้างต่อ" = vendor lock-in

[expand: handover checklist + documentation standard]

## 3. 7 คำถามที่ควรถามก่อนจ่ายงวดแรก

1. ขอดู deliverable ตัวอย่างของ client คนล่าสุดได้ไหม
2. industry ที่เคยทำขนาดธุรกิจใกล้เราที่สุดคืออะไร
3. ใครจะ deliver จริงๆ
4. payment structure แบบไหน + เหตุผล
5. ถ้าผลลัพธ์ไม่เป็นไปตามที่ตกลง คืนเงินไหม / รับผิดชอบยังไง
6. หลังจบโครงการทีมเรา maintain เองได้ในเวลากี่เดือน
7. ขอ reference call กับ client เก่าได้ไหม (1 รายก็พอ)

[expand: ทำไมแต่ละข้อสำคัญ + red flag ของคำตอบที่ "ไม่ใช่"]

## 4. Red flags ที่ควรเดินจาก

- **"การันตีเพิ่มยอด X%"** — ไม่มีใครการันตีได้จริง
- **เรียกค่าตัวรายชั่วโมงไม่มี scope** — เปิดประตูเก็บเงินไม่จบ
- **ไม่ขอดูข้อมูลทีม / sales pipeline ก่อนเสนอ** — generic solution
- **ใช้ buzzword ตอบทุกคำถาม** — "ขับเคลื่อน", "ครบวงจร", "ตอบโจทย์ทุก..."
- **case study เป็น brand ใหญ่ทั้งหมด** — กับ SME คุณ relevance เป็นศูนย์

## 5. Decision matrix — เลือกตามสถานการณ์

| สถานการณ์ของคุณ | ควรเลือก |
|---|---|
| ทีมขาย run ได้แล้ว แค่อยากปรับ skill | Trainer ฿20-50k/วัน |
| ผู้บริหารคนเดียวต้องการ sounding board | Coach ฿15-30k/เดือน |
| ทีมไม่ run เอง / ไม่มีระบบ KPI / ยอดผูกกับเจ้าของ | Consultant (project-based) ฿50-200k/project |
| อยากทดลองก่อน commit ใหญ่ | Workshop 1 วัน ฿20-30k |

---

## 6. ถ้าเลือกไม่ถูก: framework "ลองเล็กก่อน"

ก่อนจ่ายค่า project ฿100k+:

1. ขอ workshop 1 วัน ราคา ฿20-30k — เห็นวิธีทำงานจริง
2. ระหว่าง workshop วัด: เขา listen ก่อน prescribe หรือไม่ / deliverable วันนั้นใช้ได้จริงไหม
3. ถ้าผ่าน 2 ข้อนี้ค่อย scope project ใหญ่
