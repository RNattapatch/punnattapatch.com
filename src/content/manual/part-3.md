## Phase 3 Operate — 3 วิธีใช้ทีม AI ของคุณ

เมื่อคุณสร้างทีม AI Agent เสร็จแล้ว คำถามต่อไปคือ จะสั่งงานพวกมันยังไงให้มีประสิทธิภาพที่สุด

วิธีการใช้งาน agent มี 3 รูปแบบหลักๆ คุณไม่จำเป็นต้องเลือกแค่วิธีเดียว ส่วนใหญ่เราจะใช้ผสมกันตามสถานการณ์

### 1. Direct Pattern: สั่งงานตรงตัวต่อตัว

วิธีที่ง่ายและตรงไปตรงมาที่สุด เหมือนคุณเดินไปคุยกับพนักงานคนหนึ่งโดยเฉพาะ

คุณ mention ไฟล์ agent ที่ต้องการขึ้นมา เช่น `@agent-sales-proposal-drafter.md` แล้วต่อด้วยคำสั่งของคุณ

**เหมาะสำหรับ:** งานที่ชัดเจน มีเจ้าภาพคนเดียว ไม่ซับซ้อน

**ตัวอย่าง:**
- "ช่วยร่าง proposal สำหรับลูกค้า A โดยใช้ข้อมูลจากอีเมลล่าสุด" (สั่ง @`proposal-drafter`)
- "สรุปประเด็นสำคัญจากบทความเกี่ยวกับเทรนด์อุตสาหกรรมนี้ให้หน่อย" (สั่ง @`market-researcher`)
- "เขียนอีเมลติดตามลูกค้า B ที่เคยคุยกันเมื่อ 2 สัปดาห์ก่อน" (สั่ง @`sales-assistant`)

### 2. Leader Pattern: สั่งงานผ่านหัวหน้าทีม

วิธีนี้คือการสร้าง agent พิเศษขึ้นมาอีก 1 ตัว ทำหน้าที่เป็น "หัวหน้าทีม" หรือ "Leader" โดยเฉพาะ

Leader agent จะมี prompt ที่บอกว่า "หน้าที่ของคุณคือรับคำสั่ง แล้ววิเคราะห์ว่าควรส่งต่องานนี้ให้ agent ตัวไหนดีที่สุด" ใน prompt ของ Leader จะมีลิสต์รายชื่อ agent ทั้งหมดในทีม พร้อมคำอธิบายสั้นๆ ว่าแต่ละตัวทำอะไรได้บ้าง

ตัวอย่างการใช้ Leader: Sale-Leader-Agent มี List รายชื่อ Agent ในทีมดังต่อไปนี้
- @sale-crm-bot มีหน้าที่ดึงข้อมูลลูกค้า
- @persona-bot มีหน้าที่เอาข้อมูลของลูกค้า มาสวมบทบาทเป็นลูกค้า แล้วให้เซลล์ใช้ฝึกการปิดการขาย
- @sale-strategy-bot มีหน้าที่ช่วยคิดกลยุทธ์การปิดการขาย

เวลาคุณจะสั่งงาน แทนที่จะไปหา agent โดยตรง คุณจะคุยกับ Leader ตัวเดียว แล้วมันจะสร้าง prompt ที่สมบูรณ์เพื่อไปสั่งงาน agent ลูกทีมที่เหมาะสมให้เอง

**เหมาะสำหรับ:** งานที่ต้องใช้ความสามารถเฉพาะทาง แต่คุณไม่แน่ใจว่าใครทำได้ดีที่สุด

**ตัวอย่าง:**
- คุณสั่ง Leader: "เตรียมข้อมูลก่อนประชุมกับลูกค้ารายใหญ่พรุ่งนี้"
- Leader วิเคราะห์: งานนี้ต้องใช้ข้อมูลการขายเก่า (สั่ง `sales-analyst`) + ข้อมูลคู่แข่ง (สั่ง `market-researcher`) + ร่าง agenda การประชุม (สั่ง `executive-assistant`)
- Leader ไปสั่งงาน Agent ลูกน้องตัวอื่นๆ ในทีมให้ทำงานแทนคุณเอง

### 3. Group Pattern: สั่งงานแบบประชุมทีม

วิธีที่ซับซ้อนขึ้น แต่ทรงพลังมากสำหรับงานที่ต้องการมุมมองหลากหลาย เหมือนการเรียกประชุมทีม (ข้อเสียคือเปลือง Token มากขึ้น)

คุณจะสร้าง prompt ใหม่ขึ้นมาหนึ่งชุด โดยเอา prompt ของ agent หลายๆ ตัวมารวมกันในหน้าต่างแชตเดียว แล้วสั่งงานครั้งเดียว

**เหมาะสำหรับ:** งานที่ต้องการระดมสมอง การวิเคราะห์ที่ซับซ้อน การหาไอเดียใหม่ๆ

**ตัวอย่าง:**
- คุณต้องการหาทางเพิ่มยอดขายในไตรมาสหน้า
- copy prompt ของ `sales-strategist`, `market-researcher`, และ `creative-copywriter` มารวมกัน
- สั่งว่า: "วิเคราะห์สถานการณ์ปัจจุบันและเสนอแคมเปญการตลาด B2B 3 แคมเปญเพื่อเพิ่มยอดขาย Q3"

Claude จะเข้าใจว่ามี "ผู้เชี่ยวชาญ" 3 คนอยู่ในห้อง และจะให้คำตอบที่แต่ละ "คน" ช่วยกันคิด ทำให้ได้ผลลัพธ์ที่ลึกและรอบด้านกว่าการสั่งงานทีละตัว

เริ่มจาก Direct Pattern ให้คล่อง แล้วค่อยๆ ลองใช้ Leader และ Group เมื่อคุณเริ่มคุ้นเคยกับทีม AI มากขึ้น

## Phase 3 Operate — Worksheet: แผนผังงานประจำสัปดาห์และเจ้าของงาน

เป้าหมายคือการเปลี่ยนจาก "การทดลองใช้ AI" ไปสู่ "การมีทีม AI ทำงานให้" อย่างเป็นระบบ

**ขั้นตอนที่ 1: ลิสต์งานที่คุณทำใน 1 สัปดาห์**

ใช้เวลา 15 นาที ลิสต์งาน recurring tasks ที่คุณทำเป็นประจำทุกสัปดาห์

**ขั้นตอนที่ 2: จับคู่งานกับ Agent**

| งานประจำสัปดาห์ | เจ้าของงาน (Agent Owner) | สถานะ |
| :--- | :--- | :--- |
| ร่างใบเสนอราคาสำหรับลูกค้าใหม่ | `proposal-drafter` | มอบหมายแล้ว |
| เขียนโพสต์ LinkedIn เกี่ยวกับ case study | `linkedin-writer` | มอบหมายแล้ว |
| หาข้อมูล 5 บริษัทเป้าหมาย | `market-researcher` | มอบหมายแล้ว |
| สรุปรายงานยอดขายประจำสัปดาห์ | `sales-analyst` | ยังไม่มี Agent |

**ขั้นตอนที่ 3: ระบุช่องว่าง**

- **งานที่ยังไม่มี agent:** สัญญาณว่าต้องสร้าง agent ตัวใหม่
- **agent ตัวเดียวรับภาระหนักเกินไป:** พิจารณาแตก agent ออกเป็น 2-3 ตัวที่เชี่ยวชาญเฉพาะทางมากขึ้น

## Phase 3 Operate — Debug Log และวิธีแก้ปัญหา "Agent Drift"

เมื่อคุณเริ่มใช้ AI Agent ไปสักพัก คุณจะเจอปรากฏการณ์ที่ผมเรียกว่า "Agent Drift"

Agent Drift คืออาการที่ agent ซึ่งเคยทำงานได้ดีเยี่ยม อยู่ๆ ก็เริ่มให้ผลลัพธ์ที่เพี้ยนไปจากเดิม เช่น ตอบไม่ตรง format, ลืม context สำคัญ

ปัญหานี้เป็นเรื่องปกติของการทำงานกับ AI ครับ ไม่ใช่ความผิดของคุณหรือระบบ AI

### `debug-log.md` — เก็บไว้ใช้ตลอดอนาคต

ทุกครั้งที่ agent ทำงานผิดพลาด เปิดไฟล์ `debug-log.md` ขึ้นมาบันทึกทันที

```markdown
### DEBUG LOG

**Date:** 2026-07-15
**Agent Name:** `proposal-drafter`
**Problem:** ให้ราคาใน proposal เป็นสกุลเงิน USD ทั้งที่สั่งให้ใช้ THB

**Fix Applied:**
- เพิ่ม constraint ใน prompt: "CRITICAL: All financial figures must be in Thai Baht (THB) only."
- ทดสอบซ้ำ ผลลัพธ์ถูกต้อง
```

### 3 รูปแบบ Agent Drift ที่พบบ่อยและวิธีแก้

**1. Context Amnesia (อาการลืมบริบท)**
- อาการ: Agent ลืมข้อมูลสำคัญที่คุณใส่ไว้ใน Business Context
- วิธีแก้: ย้ายข้อมูลที่สำคัญไปไว้ท้ายสุดของ prompt หรือใช้ `**CRITICAL CONTEXT:** Our company name is [Your Company Name].`

**2. Format Rebellion (อาการต่อต้าน Format)**
- อาการ: คุณสั่งให้ตอบเป็น bullet point แต่ AI ตอบเป็นย่อหน้ายาวๆ
- วิธีแก้: ใช้ "Few-shot Prompting" ให้ตัวอย่างผลลัพธ์ที่ถูกต้องไปใน prompt เลย

**3. Verbosity Fluctuation (อาการพูดมาก/น้อยผิดปกติ)**
- อาการ: Agent ที่เคยสรุปเก่งๆ อยู่ๆ ก็เขียนมายาว 3 หน้า
- วิธีแก้: เพิ่ม constraint เกี่ยวกับความยาวที่ชัดเจน เช่น `Provide a summary in 3 bullet points, no more than 20 words per point.`

## Phase 4 Scale — Math of Volume และจุดที่ควรขยับไปใช้ n8n Automation

เมื่อคุณใช้ทีม AI Agent จนคล่องแล้ว คุณจะเริ่มเห็น "มูลค่า" ที่แท้จริงของมัน ซึ่งคำนวณออกมาเป็นตัวเลขได้

**สูตรคำนวณ ROI ง่ายๆ**

(ชั่วโมงที่ประหยัดได้ต่อสัปดาห์) × (50 สัปดาห์ทำงานต่อปี) × (ค่าแรงเฉลี่ยต่อชั่วโมง) = มูลค่าที่ประหยัดได้ต่อปี

- **TIER 1: The Solopreneur / Manager** — ประหยัดเวลา 5 ชม./สัปดาห์ → **฿125,000 ต่อปี**
- **TIER 2: The Small Team** — ประหยัดเวลา 80 ชม./สัปดาห์ → **฿2,000,000 ต่อปี**
- **TIER 3: The Organization** — ประหยัดเวลา 240 ชม./สัปดาห์ → **฿6,000,000 ต่อปี**

### สัญญาณที่บอกว่าคุณต้อง Scale ด้วย Automation

ถ้าคุณพบว่าตัวเองหรือทีมงานกำลัง...
- ใช้เวลามากกว่า 1-2 ชั่วโมงต่อวันไปกับการ copy-paste ระหว่างระบบต่างๆ กับ Claude
- เริ่มทำงานพลาดเพราะ copy ผิดหน้าต่าง
- มีความคิดว่า "ถ้ามันกดปุ่มเดียวแล้วทำงานอัตโนมัติได้เลยก็คงดี"

นั่นคือสัญญาณว่าคุณพร้อมจะขยับไปสู่ขั้นต่อไป: **Workflow Automation**

### Wedge-First Approach to Automation

มองหางานที่มีลักษณะครบ 3 ข้อนี้:
1. **High Volume** — งานที่ทำซ้ำๆ บ่อยที่สุด
2. **High Value** — ถ้าทำเร็วขึ้นหรือผิดพลาดน้อยลง จะส่งผลกระทบต่อธุรกิจสูง
3. **Simple Trigger** — เริ่มต้นด้วย trigger ง่ายๆ เช่น "มีอีเมลใหม่เข้ามา" หรือ "ถึงเวลา 9 โมงเช้า"

เครื่องมือที่ผมแนะนำให้เริ่มศึกษาคือ **n8n.io** — เครื่องมือ open-source ที่สร้าง workflow อัตโนมัติด้วยการลากวาง (visual interface) เชื่อมต่อกับ Claude และระบบอื่นๆ เช่น Gmail, Google Sheets, Slack

## Phase 4 Scale — เมื่อไหร่ที่ควร "จบการศึกษา" ไปหาผู้เชี่ยวชาญ

Kit นี้เหมาะที่สุดสำหรับทีม 5-20 คน แต่มีขอบเขตของมัน

**5 สัญญาณที่คุณต้องการผู้เชี่ยวชาญ:**

1. **ทีมของคุณมีขนาดใหญ่กว่า 20 คน** — ต้องการโครงสร้างการทำงาน (governance) ที่ออกแบบมาโดยเฉพาะ
2. **คุณต้องการเชื่อมต่อกับ ERP / CRM ที่เป็น custom-built** — ต้องทำงานกับ API ที่ซับซ้อน
3. **คุณต้องการให้ AI ทำงานสำคัญแบบ "อัตโนมัติ 100%"** — ต้องการการออกแบบสถาปัตยกรรมระบบและการจัดการความเสี่ยงที่เข้มข้นกว่ามาก
4. **งานที่คุณต้องการเกี่ยวข้องกับข้อมูลที่มีความอ่อนไหวสูง** — ข้อมูลทางการเงิน ข้อมูลสุขภาพ ความลับทางการค้า
5. **คุณเห็นศักยภาพ แต่ไม่มีเวลาลงมือทำเอง** — การจ้างผู้เชี่ยวชาญทำ workshop แบบเร่งรัด 1-2 วัน จะเป็นการใช้เงินซื้อเวลาที่คุ้มค่าที่สุด

**ทางเลือกสำหรับก้าวต่อไป:**
- **เจ้าของธุรกิจ (Owner path):** [In-house Workshop → punnattapatch.com/services/ai-workshop](https://punnattapatch.com/services/ai-workshop)
- **ผู้จัดการทีม (Manager path):** [Public Training P1 → punnattapatch.com/training/p1-ai-workshop-1-day](https://punnattapatch.com/training/p1-ai-workshop-1-day)

## Phase 4 Scale — MCP + Connectors (ทีม AI ทำงานแทนเลขาได้จริง)

ถ้าใช้ Claude Code ไป 1-2 สัปดาห์ คุณจะเริ่มอึดอัด

agent ฉลาดมาก แต่อยู่ในกล่อง ขอให้สร้าง Canva slide → ทำไม่ได้ ขอให้ดึง email ลูกค้าที่ทักมา 7 วันที่แล้ว → ทำไม่ได้ ขอให้เช็คตารางว่างของคุณก่อนนัดประชุม → ทำไม่ได้

ทำไม? เพราะ AI ตัวเปล่ายังไม่มี "มือ" ไปจับ tools รอบตัวคุณได้

นี่คือจุดที่ **MCP** เข้ามาแก้

### MCP คืออะไร — เลขา AI ที่มีกุญแจทุกบาน

MCP ย่อมาจาก **Model Context Protocol** เป็นมาตรฐานที่ Anthropic เปิดให้ AI "พูด" กับโปรแกรมข้างนอกได้

เปรียบเทียบให้เห็นภาพ:
- เลขาที่ไม่มี MCP = พูดเก่ง คิดเก่ง แต่ไม่มีกุญแจห้องอะไรเลย ทำได้แค่บอกคุณว่าควรทำอะไร คุณต้องลงมือเอง
- เลขาที่มี MCP = พูดเก่ง คิดเก่ง มีกุญแจห้อง Canva ห้อง Google Drive ห้อง Gmail ห้อง Calendar บอกแล้วทำให้เลย ไม่ต้องสั่งให้คุณทำเอง

MCP คือกุญแจชุดนั้น แต่ละ "MCP server" = กุญแจของห้องหนึ่ง

### MCP Server ที่พร้อมใช้แล้วตอนนี้ (เม.ย. 2026)

ติดตั้งใน Claude Desktop App ในเมนู `Settings → Connectors`

| MCP Server | ทำอะไรได้ | ตัวอย่างคำสั่ง |
|---|---|---|
| **Canva** | สร้าง slide / poster / infographic / export PDF | "ทำ slide pitch deck ลูกค้า X 8 หน้า ใช้ template Modern Blue" |
| **Google Drive** | อ่าน/เขียน/ค้นไฟล์ใน Drive | "หา proposal ที่ส่งให้ลูกค้า A เดือนก่อน สรุปให้ฟังหน่อย" |
| **Gmail** | อ่าน ค้น ร่าง ส่ง email จัดการ label | "หา email จากคุณสมชายที่ทักมา 7 วันที่แล้ว ร่าง follow-up reply" |
| **Google Calendar** | ดูตารางว่าง นัดประชุม เลื่อนนัด เตือนล่วงหน้า | "เช็คเวลาว่างผมพรุ่งนี้ นัดคุณ B 1 ชม. ส่ง invite" |
| **n8n** | trigger workflow automation ที่คุณสร้างไว้ใน n8n cloud | "รัน workflow weekly-report-gen ส่งผลให้ผมหลังเสร็จ" |

### ติดตั้ง MCP — 5 นาที / ตัว

1. เปิด Claude Desktop App
2. ไป `Settings → Connectors` (Mac: `⌘+,` → tab Connectors)
3. คลิก connector ที่ต้องการ (เช่น Gmail)
4. กด **Connect** → จะเปิด browser ให้ sign in Google account
5. กรอก permission scope → Allow
6. กลับมาที่ App → connector เปลี่ยนสถานะเป็น "Connected"

### กฎทอง 3 ข้อตอนใช้ MCP

1. **เริ่มจาก read-only ก่อน** — ครั้งแรกที่ลอง MCP ตัวใหม่ อย่าให้ permission write ทันที ให้ดูแค่ "อ่าน" ก่อน ใช้ 1-2 อาทิตย์ดูว่า agent เข้าใจระบบของคุณก่อน แล้วค่อย upgrade เป็น write
2. **ห้ามต่อ MCP เข้า production database โดยไม่มี approval gate** — ตั้ง Constraint ใน agent file: "ทุกคำสั่ง write ต้องถามผู้ใช้ confirm ก่อน"
3. **เก็บ revoke plan** — ทุก MCP ต้องมีวิธีถอดทันที ใน `Settings → Connectors → [tool] → Disconnect`
